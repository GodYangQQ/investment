# 股票分析 Prompt 模板

---

## ⚠️ AI 工作流（必须严格按顺序执行）

> **股票池来源**：所有股票的代码列表统一存放在 `data/my_pool.csv`（每行一个6位代码）。任何涉及选股、排名、批量分析的操作都必须基于此文件，不要使用其他自选股列表或手动指定代码。

### Step 0: 数据质量门禁（所有场景必须先执行）

> ⚠️ **在任何打分、选股、分析操作之前，AI 必须先检查数据质量。数据有问题 → 立即停止，提示用户修复，禁止继续。**

AI 必须在每个 Step 执行后检查输出，发现以下任一异常即 **终止后续所有步骤**：

#### 0.1 脚本运行异常检测

| 异常类型 | 检测方式 | 处理 |
|----------|---------|------|
| **脚本报错** | 终端输出含 `Error`、`Traceback`、非零 exit code | 🛑 停止。把错误信息展示给用户，提示修复 |
| **网络超时** | 含 `timeout`、`ConnectionError`、`HTTPError` | 🛑 停止。提示用户检查网络/API可用性 |
| **大量 Warning** | 同一 Warning 出现 ≥10 次 | 🛑 停止。说明可能是 API 变更，提示检查对应脚本 |
| **API 返回空** | `无法获取`、`No data`、返回0行 | 🛑 停止。提示用户确认股票代码正确/数据源可用 |

#### 0.2 输出数据完整性检测

| 异常类型 | 检测方式 | 处理 |
|----------|---------|------|
| **关键列为空** | ROE、毛利率、PE_TTM 全为空或 >50%为空 | 🛑 停止。提示 `akshare` API 可能已变更，需更新 `fundamental_filter.py` |
| **总得分异常** | 所有股票总得分相同或全为0 | 🛑 停止。提示打分逻辑可能有 bug |
| **数据量异常** | 输出 CSV 行数 < 预期20%（如 Top100 只出了 5 条） | 🛑 停止。提示数据源可能受限 |
| **日期陈旧** | 财报数据日期距今 >6个月 | ⚠️ 警告用户数据可能过时，但仍可继续 |
| **PE 值异常** | PE_TTM 出现 0、负数占比 >30% | ⚠️ 警告用户但继续（A股亏损股多属正常），基本面过滤会自动处理 |

#### 0.3 检测到异常时的 AI 响应模板

```
⚠️ 数据异常，停止分析。

问题: [具体异常描述]
来源: [哪个脚本/哪个输出文件]
建议修复: [具体修复方向]

❌ 在修复前不继续选股/打分操作。
```

### 场景 A：全市场扫描选股（日常使用）

当用户说"跑一遍排名""帮我选股""看看今天哪些值得买"时，按以下顺序执行：

```
Step 1: 全市场量化排名
  运行: python run/rank.py --top 100 --deep-check-top 300
  输入: 全A股（自动获取）
  输出: output/quant_top100.csv（含技术打分+基本面过滤+仓位乘数）
  说明: --keep-markets "" 不过滤板块（默认排除创业/科创），--no-fundamental-filter 禁用基本面过滤
  ⚠️ 运行后立即检查: 终端有无报错？CSV是否正常生成？行数≥80？

Step 2: AI 读取排名结果
  读取: output/quant_top100.csv
  关注列: 排名, 代码, 名称, 总得分, 可买度, PE_TTM, ROE, 毛利率, 仓位乘数, 基本面原因
  ⚠️ 检查: ROE/毛利率列是否超过50%为空？总得分是否有正常分布？
  如果数据异常 → 执行 Step 0 的停止流程
  
Step 3: AI 筛选候选标的
  从 Top100 中筛选:
    - 基本面通过=True
    - 可买度 ∈ {🟢 强烈推荐, 🟢 推荐}（即总得分≥68）
    - 仓位乘数 ≥ 0.5
  按"总得分×仓位乘数"降序排列，取前10-20只作为候选

Step 4: 对每只候选股深度分析
  对每只候选股运行: python core/stock_strategy.py <代码>
  **搜索近期新闻**：对每只候选股，搜索并整理最近1周消息面（券商研报/公司公告/行业新闻）
  将输出粘贴到下面的"单股分析 Prompt 模板"中，发给 AI 逐只分析
  
Step 5: AI 输出最终交易清单
  汇总所有深度分析结果，按"基本面⭐ + 概念⭐ + 动量"排序
  输出包含: 买入价, 止盈价, 止损价, 建议仓位%
```

### 场景 B：对指定股票池批量分析

当用户说"分析一下我的股票池""看看持仓哪些该动"时：

```
Step 1: 读取股票池
  读取: data/my_pool.csv（格式: 代码,名称,层级,成本价,持仓量,买入日期）
  ⚠️ 检查: 文件是否存在？代码数量>0？代码格式是否6位数字？

Step 2: 批量量化打分（唯一推荐方式）
  运行: python core/quant_score.py --pool data/my_pool.csv --output output/pool_scores_YYYYMMDD.json
  ⚠️ 这是唯一正确的方式，不要逐只运行 stock_strategy.py（太慢，181只需要~15分钟）。
  ⚠️ 也不要用 run_rank.py（那是全市场扫描，不是针对股票池的）。
  ⚠️ 预计耗时: 181只 × ~2秒/只 ≈ 6分钟。等待完成即可，不要中途终止。
  ⚠️ 检查: 终端输出每只股票的评分是否正常？

Step 3: AI 汇总排名
  从终端输出中提取每只股票的总得分、可买度，按总得分降序排列。
  筛选总分≥68（🟢 推荐及以上）的股票作为候选。

Step 4: 对候选股深度分析（仅Top 5-10）
  只对排名前5-10的候选股运行: python core/stock_strategy.py <代码>
  **搜索近期新闻**：对每只候选股，搜索并整理最近1周消息面
  将输出粘贴到下面的"单股分析 Prompt 模板"中，逐只分析。
  ⚠️ 不要对所有181只都做深度分析！只分析Top候选。

Step 5: AI 输出最终交易清单
  汇总: 排名 | 股票 | 操作(买/卖/持) | 买入价 | 止盈 | 止损 | 仓位%
```

### 场景 C：单股深度分析

当用户说"分析一下某某股票"时：

```
Step 1: 获取基本面+技术面数据
  运行: python core/stock_strategy.py <代码>
  如果用户提供了成本价+持仓，带上 --cost --shares --buy-date 参数运行 quant_score.py
  ⚠️ 检查: 脚本是否正常输出？报错则停止

Step 2: 搜索近期新闻（必须执行，禁止跳过）
  ⚠️ **硬性要求：禁止在消息面章节写"无消息"或留空。必须逐一搜索以下源，**
  ⚠️ **直到找到至少2条有效新闻为止。只有所有源都无结果才能写"近期无重大消息面变化"。**

  使用 WebFetch 按以下顺序搜索（优先排名靠前的，至少覆盖3个源）:

  ① 新浪财经个股页（必搜，编码GBK，页面底部有"公司新闻"和"研究报告"列表）
     URL: https://finance.sina.com.cn/realstock/company/sh{代码}/nc.shtml  (沪市 sh)
     URL: https://finance.sina.com.cn/realstock/company/sz{代码}/nc.shtml  (深市 sz)
     提取: 公司新闻列表(含日期和标题) + 券商研报列表(含评级和目标价)

  ② 东方财富新闻搜索（可选，聚合新闻搜索，结构简单容易解析）
     URL: https://so.eastmoney.com/news/s?keyword={股票名称或代码}
     提取: 按日期排列的新闻列表

  ③ 雪球个股页（可选，页面不含JS也能拿到部分结构化数据）
     URL: https://xueqiu.com/S/SH{代码}  (沪市)
     URL: https://xueqiu.com/S/SZ{代码}  (深市)
     提取: 公司简介 + 关注人数 + 关键财务指标(PE/PB/ROE/股息率)

  ④ 巨潮资讯（可选，最权威的官方公告源）
     URL: http://www.cninfo.com.cn/new/disclosure/stock?stockCode={代码}&orgId={orgId}
     提取: 公司最新公告标题和日期

  ❌ 以下源需要JS渲染，不可用 WebFetch：财联社(cls.cn)、同花顺(10jqka.com)、华尔街见闻(wallstreetcn.com)

  提取信息:
    - 最近3天公司新闻/公告（利好/利空）
    - 最近1个月券商研报观点及目标价
    - 板块/行业重大政策或事件（如"PCB板块涨停潮""英伟达Rubin发布"等）

Step 3: 汇总消息面判断
  消息面分两层覆盖，缺一不可：

  **① 公司层面**：从Step 2搜索结果中提取，整理为表格:
     | 日期 | 来源 | 内容摘要 | 方向(✅利好/⚠️中性/❌利空) |
  判断标准:
     - ✅利好: 券商上调评级/目标价、订单/业绩超预期、行业政策支持、板块集体走强
     - ⚠️中性: 例行公告、股东增减持<1%、常规研报维持评级
     - ❌利空: 减持>3%、业绩预警、监管处罚(关注函/警示函)、行业政策收紧、大股东质押风险、被官媒点名"妖股"、连续一字跌停

  **② 行业/板块层面（必须补充）**：额外搜索一次行业关键词，覆盖:
     - 所属行业景气度新闻（如"PCB行业景气度""索具制造出口""玻璃基板产能"）
     - 上下游动态（如"CCL涨价对PCB影响""玻纤供需"）
     - 板块资金流向（如"PCB板块今日涨停潮""军工板块集体回调"）
     - 行业政策变化（如"出口退税调整""美国关税""环保限产"）
  在消息面表格末尾追加行业层面条目，标注为"[行业]"。

  **③ 宏观/地缘层面（如涉及必须补充）**：额外搜索一次宏观关键词，覆盖:
     - 地缘政治事件（如"红海局势""中东冲突""台海""俄乌"）
     - 关税/贸易政策（如"美国对华关税""出口管制""芯片制裁"）
     - 汇率/大宗商品（如"人民币汇率""原油价格""铜价暴涨"）
     - 全球流动性（如"美联储利率决议""美债收益率""美元指数"）
  在消息面表格末尾追加宏观层面条目，标注为"[宏观]"。
  ⚠️ 对于航运/能源/有色/出口型企业，宏观/地缘层面必须覆盖，不可跳过。

  **④ A股市场层面（必须覆盖）**：额外搜索一次A股大盘关键词，覆盖:
     - 大盘走势与成交量（如"A股成交额破2万亿""上证突破4100""北向资金大幅流入/流出"）
     - 政策/监管动态（如"证监会新规""降准降息""印花税调整""国常会稳市场"）
     - 市场情绪/风格（如"中小盘活跃""权重股护盘""AI概念全线爆发""资金从新能源流向AI"）
     - 资金面（如"两融余额变化""北向资金动向""ETF资金流向"）
  在消息面表格末尾追加A股大盘层面条目，标注为"[A股]"。

  综合判断: 分别总结公司层面+行业层面+宏观/地缘层面+A股大盘层面的消息面影响方向及强度

Step 4: AI 按"单股分析 Prompt 模板"逐章节输出
  必须包含: 消息面分析, 基本面快评, 双维度评分, 量化动量因子, 形态分析, 止盈止损表
```

## 使用方法（手动操作指南）

1. 运行脚本获取**最新盘中/收盘数据**：
```bash
# 先获取大盘数据
python3 -c "
import requests, re
r = requests.get('https://qt.gtimg.cn/q=sh000001', timeout=10)
r.encoding = 'gbk'
m = re.search(r'=\"(.+?)\"', r.text)
f = m.group(1).split('~')
print(f'上证指数: {f[3]} ({float(f[32]):+.2f}%)  |  成交额: {int(float(f[37]))}亿')
"

# 再获取个股数据
python core/stock_strategy.py <股票代码>
```
   - 交易时段运行 → 获取实时盘中数据
   - 收盘后运行 → 获取当日完整数据
   - 注意盘中数据未收盘，仅作参考

2. 复制脚本输出的全部结构化数据，粘贴到 Prompt 中的 `{数据}` 位置，发给大模型。

3. **盘中/盘后数据差异**：
   - **15:00 之前** → 当日量能数据不全（仅部分时段），勿据此下"极致缩量"等结论；应侧重实时价格走势、上下影线和振幅
   - **15:00 收盘后** → 量价数据完整，可正常分析

4. **双轨制仓位管理**：
   - **波段轨（70%资金）**：按止盈止损表固定价位操作，赚取波动收益，可反复做一只票
   - **趋势轨（30%资金）**：同价位建仓但不设止盈，MA60/回撤-25%才离场，用于捕捉十倍股

## 输出格式（严格遵循）

### ⚠️ 分析权重原则
> **决策权重排序**（按重要性）：
> 1. **基本面（60%）**：ROE、现金流/净利润比、估值(PE) → 决定**买不买、卖不卖**
> 2. **量价关系（25%）**：Chaikin资金流、量比、量价相关性 → 决定**时机**
> 3. **RSI+形态（15%）**：仅作辅助确认，不可独立决策
>
> **三要素冲突时，高权重优先。基本面不达标 = 不买，不管技术面多好看。**

### ⚠️ 基本面硬过滤与降权（必读，不可跳过）
> **量化排名的高分股 ≠ 值得买。** 大量技术面高分股存在基本面硬伤，必须在分析阶段剔除或降权。

#### 一票否决（以下任一命中 → ❌ 不买，直接淘汰）
> ⚠️ **亏损股按分级处理，不一刀切。**

| 条件 | 标准 | 原因 |
|------|------|------|
| PE离谱 | PE-TTM > 200 | 估值虚高，泡沫风险 |
| 负债率极高 | 资产负债率 > 85% | 财务风险大（银行/保险除外） |

#### 亏损股分级处理
> **高毛利+高增长的亏损企业可能是下一个十倍股。**

| 等级 | 条件 | 仓位 | 说明 |
|------|------|------|------|
| ❌ 真垃圾 | 亏损 + 营收增速<-10% | **排除** | 没增长前景 |
| ⚠️ 亏损潜力股 | 亏损 + 毛利率>30% + 营收增速>20% | **×0.3** | 高毛利高增长，仅趋势轨持有，不设止盈 |
| 🟡 亏损待观察 | 其他亏损情况 | **×0.5** | 需人工判断 |

> **亏损潜力股操作规则**：仅趋势轨、不设止盈、趋势止损MA60或回撤-25%、单只仓位上限4.5%、亏损股总持仓≤15%。

#### 降权机制（未命中否决，但指标偏弱 → 降低仓位）

> ⚠️ **现金流为负不等于造假**：高成长企业（营收增速>20%）现金流为负通常是扩产/铺货所致，毛利率>30%+净利增速>30%时视为良性扩张，不降权。

| 条件 | 标准 | 降权 | 说明 |
|------|------|------|------|
| ROE偏低 | 4% ≤ ROE < 8% | 仓位×0.6 | 周期股×0.8 |
| 现金流偏弱 | 0 ≤ 现金流/净利 < 0.5 | 仓位×0.7 | |
| 现金流为负（良性） | 现金流/净利 < 0，但毛利率>30% + 营收增速>20% | **不降权** | 扩产/铺货期，利润质量高即可 |
| 现金流为负（警示） | 现金流/净利 < 0，毛利率≤30% 或 营收增速≤20% | 仓位×0.5 | 可能赊销严重或回款困难 |
| PE偏高 | 60 < PE ≤ 200 | 仓位×0.7 | 成长股×0.9 |

> **使用筛选脚本获取基本面数据**：
> ```bash
> # 全市场排名 + 基本面过滤（推荐）
> python run/rank.py                            # 默认: 排除创业板+科创板 + 基本面分级过滤
> python run/rank.py --keep-markets "主板"       # 只要主板
> python run/rank.py --no-fundamental-filter    # 仅技术面排名（不推荐，大量垃圾）
> python run/rank.py --deep-check-top 300       # 深度检查前300只
> ```
> 脚本会自动获取ROE、毛利率、营收增速、现金流/净利、负债率并做分级处理。
> AI 读取输出CSV中的"基本面原因"列和"仓位乘数"列即可。
> 对于降权项（未被硬过滤的），AI 必须在仓位决策中叠加对应系数。

### RSI 辅助参考（权重15%，不可独立决策）
> ⚠️ RSI仅作节奏辅助。买卖决策以基本面(60%)+量价(25%)为准。
> **RSI不含量**：RSI 80+放量=真买盘可续涨，RSI 80+缩量=假突破随时反杀。强趋势可长期"超买"。

**RSI使用原则（必须与量确认）**：
- RSI>80且放量↑ → 强趋势延续，不急于卖
- RSI>80且缩量↓ → 见顶预警，减仓
- RSI<30且放量↑ → 超跌反弹确认，可买
- RSI<30且缩量↓ → 阴跌未止，不抄底
- 止损以价格破位+放量为准，RSI超卖不作"不割"的理由

> **趋势轨综合决策**（基本面 + 概念 + 动量）：
> - 基本面≥4 + 概念≥4 + 动量≥5 → 趋势轨 **30%**（A级，十倍股候选）
> - 基本面≥4 + 概念≥4 + 动量3-4 → 趋势轨 **25%**（短期回调=长线买点）
> - 基本面≥4 + 概念≤3 + 动量≥5 → 趋势轨 **20%**（好公司强动量）
> - 基本面≥4 + 概念≤3 + 动量≤4 → 趋势轨 **15%**（稳健压舱石）
> - 基本面≤3 + 概念≥4 → **不买**，列入观察清单
> - 基本面≤3 + 概念≤3 → 不看

> **ATR（Average True Range，平均真实波幅）说明**：衡量股票日均波动幅度的技术指标。
> - ATR(14)=当前值，表示近14日平均每天波动约 **ATR/股价 ×100%**
> - 第二止盈的 **ATR×2 移动止盈** = 从突破后的最高点回撤 2个ATR 即卖出
> - 例：ATR=6、最高涨到120，则移动止盈线 = 120-12 = 108，价格跌破108即卖
> - 当前ATR值在量化评分表中可查到
> - **当股价未突破第一止盈时**，第二止盈线 = 已有最高价 - ATR×2，通常与止损位接近（属于正常，突破后才有效拉开）

> 回撤止盈按**基本面+概念**分档，不搞一刀切。好公司容忍更大波动。
>
> **止盈原则：止盈价必须是"市场会在那里犹豫的真实价格"**——布林上轨、60日高、前密集成交区。
> 固定百分比（+15%/+25%）只在合理范围内做参考，不能替代真实技术位。
> 例：若最近阻力距现价仅+8%、ATR每日5%，则第一止盈就设在这个阻力位（+8%），
> 不硬凑15%。反之若阻力在+30%开外，先吃最近一段，突破后再移动止盈跟随。
>
> **形态修正原则（必须执行）：**

## 量化策略：多因子打分轮动（核心决策引擎）

> **设计原则**：零未来函数。所有指标计算基于截止**前一个交易日**的完整K线数据，当日决策仅使用开盘价或实时价。不依赖聚宽平台，全部使用腾讯行情API + akshare 可获取的数据。

---

### 一、策略架构总览

```
T日 盘后/早盘 → 【多因子打分】基于T-1日及之前数据
              → 每只股票独立打分，不横向排名
              → 判定每只的"可买度"和"建议仓位"
T日 盘中      → 对比现有持仓，触发卖出/买入建议
              → 实时监控止盈止损
```

**关键区别**：不是"20只中选最好的5只"，而是"每只股票独立判断：现在值不值得买？值得几成仓位？"

---

### 二、数据时点反未来函数规范

| 数据 | 来源 | 时点硬约束 |
|------|------|-----------|
| 日K线（OHLCV） | 腾讯前复权K线(`ifzq.gtimg.cn`) | `end_date = T-1日`，**严格不含T日数据** |
| 技术指标(MA/MACD/RSI/布林/ATR/OBV/CMF等20+指标) | 基于上述K线计算 | 全部来自T-1日及之前 |
| 实时行情 | 腾讯行情(`qt.gtimg.cn`) | T日盘中执行，用`last_price`(已发生的价格) |
| 财务报表(ROE/利润增速/毛利率等) | akshare | 最新已披露季报，公开历史数据 |
| 上证指数 | `sh000001` | 大盘K线同样截止T-1日 |
| PE/PB | 腾讯行情 | T日实时值 |

> **禁止使用的数据**：聚宽内部数据(`jqdata`/`get_price`/`get_current_data`)、任何T日收盘价(盘中尚未产生)、未来季报/年报。

### 四、多因子技术打分模型（每日自动计算）

每只股票**独立打分**，满分100。所有计算基于T-1日及之前的K线数据。
**评分由 `python core/quant_score.py` 精确计算（连续值，保留1位小数），AI 直接引用结果即可。**

#### 4.1 趋势因子（满分25分，连续值）

基于 MA5/MA10/MA20 之间的间距百分比 + 价格距MA20/MA60的偏离程度，用分段线性函数计算连续得分。核心思想：多头排列且间距合理（各均线间距1-3%）得分最高，均线空头排列或间距过大得分递减。

#### 4.2 位置因子（满分25分，连续值）

核心是距60日高点回撤%，用非对称抛物线映射：回撤8-20%为最佳区间（得分15-20），<3%追高风险（得分4-5），>30%破位风险（得分递减）。同时引入距20日低点的位置作为补充。

#### 4.3 量价因子（满分20分，连续值）

综合四维量价信号：①量比（5日均量/20日均量，最佳1.0-2.5）②OBV趋势 ③CMF资金流(20日) ④量价相关系数。四个子维度各0-8/0-6/0-3/0-3分，加总后为0-20连续值。

#### 4.4 RSI因子（满分15分，连续值）

用非对称钟形函数：RSI在52附近（中心）得满分15分，向右（超买方向）缓慢衰减、向左（超卖方向）稍快衰减。70以上加速扣分，30以下加速扣分。

#### 4.5 波动率因子（满分15分，连续值）

ATR(14)/价格 的百分比映射：2-5%为最佳区间（得分13-15），<1%极低波动（得分3-7），>8%极高波动（得分递减至3以下）。±1的5日波动率微调。

#### 4.6 附加项（±13分，连续值）

① MACD柱值强度（用柱值/价格的标准化，金叉+5~7/死叉-5~-7，趋势中±1~3）
② 连续阴/阳线（连阴3天+1~3，连阳4天-1~-3）
③ 布林带位置（下轨附近+2，上轨附近-1.5，极窄<-2）

### 六、卖出规则（盘中实时检查，按标的等级分档）

> **反未来函数**：使用盘中 `last_price`（已发生的实时价格），不是收盘价。

> **设计原则**：回撤止盈按基本面+概念分档，不搞一刀切。好公司（A+/A级）容忍更大波动，捕捉十倍股；基本面弱的（B级）早割快跑。

#### 6.1 标的等级定义

| 等级 | 条件 | 回撤止盈线 | 典型标的 |
|------|------|-----------|----------|
| **A+** | 基本面≥4 + 概念≥4 | **-10%** | 瑞芯微、兆易创新、沪电股份、生益科技 |
| **A** | 基本面≥4 + 概念≤3 | **-8%** | 紫金矿业、中远海能、兴业银锡 |
| **B** | 基本面≤3 | **-6%** | 立讯精密、工业富联、天齐锂业、双环传动、恺英网络 |
| **兜底** | ATR×2 比上述值更大 | 取 ATR×2 | 防止正常波动误杀 |

#### 6.2 卖出规则

| 优先级 | 条件 | 动作 | 反未来函数保证 |
|--------|------|------|---------------|
| 1 | 盘中亏损 ≥ **10%**（vs 成本价） | **硬止损**，立即卖出 | `last_price` 是已发生的实时价 |
| 2 | 从买入后最高点回撤 ≥ 分档线（6.1） | **回撤止盈**，卖出 | `highest_price`（已记录）+`last_price` |
| — | 涨停次日保护 | **涨停次交易日不执行规则2和4**，给一天观察 | 涨停板是最强多头信号 |
| — | 错过止盈窗保护 | 曾盈利≥10%未卖现已回落至成本附近→规则2触发时**卖在成本价**而非现价 | 不因犹豫而被反杀 |
| 3 | 盈利 ≥ 15%（波段轨半仓）/ ≥ 25%（波段轨全仓） | **固定止盈**，卖出 | 趋势轨不设固定止盈 |

### 九、Agent 执行流程

> **核心原则**：数学计算全部交给 `quant_score.py` 脚本，Agent 只负责读取结果 + 结合基本面/概念做综合解读。

```
1. 一键获取大盘风控 + 所有股票量化评分
   → python core/quant_score.py --pool pool.csv --output /tmp/scores.json
   → 输出包含：大盘风控(仓位上限) + 每只股票的5因子评分 + 成本修正决策
   → AI 无需自己计算任何数值

2. 如需单只股票更详细数据（基本面/K线走势等）
   → python core/stock_strategy.py <代码> --kline-days 120
   → 输出包含：财务数据/量价关系/K线形态等定性分析素材

3. AI 综合解读
   → 读取 quant_score.py 的精确评分 → 确定买入优先级
   → 读取 stock_strategy.py 的详细数据 → 补充趋势判断、风险提示
   → 结合用户持仓表 → 给出最终操作建议
   → 叠加大盘风控 → 确认总仓位上限

4. 输出
   → 排序表（按量化总分降序）
   → 明确操作：买入xx股 / 加仓 / 减仓 / 卖出 / 持有
   → 标注风险提示
```

### 十、量化因子 vs 原有人工分析因子的关系

| 维度 | 量化多因子(第四~七章) | 原有人工分析因子(基本面⭐/概念⭐/动量) |
|------|----------------------|--------------------------------------|
| 用途 | **每日自动计算**，给出买卖时机 | **定性判断**，给出标的质量 |
| 触发 | 技术面可买度 ≥ 55 才触发买入建议 | 基本面≥4 才进入核心池 |
| 关系 | 技术面决定**何时买卖** | 基本面决定**是否值得买** |
| 决策顺序 | 第二层：择时 | 第一层：选股 |

> **最终决策 = 基本面筛选(人工) × 量化打分(自动) × 大盘风控(自动)**
> 三者全满足才建议买入，任一不满足则观望或卖出。
